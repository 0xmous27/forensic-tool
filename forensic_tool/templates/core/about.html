About Our
Forensic Tool

Developed to detect NTFS timestamp forgery using advanced digital forensic techniques — bringing scientific rigour to digital evidence investigation.

Our Story

Why We Built This

In digital forensics, timestamps are evidence. They tell investigators when a file was created, modified, or accessed — forming the backbone of any timeline reconstruction. But what happens when those timestamps are deliberately manipulated?

NTFS timestamp forgery is a real and growing threat. Attackers use tools to alter file metadata, erasing their tracks and misleading investigators. Traditional forensic workflows often lack automated, cross-source correlation — leaving analysts to manually compare $MFT entries, USN journals, event logs, and registry hives.

We built ForensicTool to close that gap. By automatically extracting artifacts from multiple NTFS sources and correlating them against each other, the system surfaces anomalies that would otherwise take hours to find — and assigns each file a forgery likelihood score backed by evidence.

Purpose

Mission & Vision

Our Mission

To provide digital forensic investigators with an automated, reliable, and scientifically grounded tool for detecting NTFS timestamp manipulation — reducing investigation time, minimising human error, and strengthening the integrity of digital evidence in legal proceedings.

Our Vision

To evolve into a comprehensive open-source forensic platform that supports multiple file systems beyond NTFS, integrates machine learning for anomaly detection, and becomes a trusted reference tool for law enforcement agencies, cybersecurity firms, and academic researchers worldwide.

Capabilities

Core Features

NTFS Artifact Extraction

Parses $MFT, $UsnJrnl, $LogFile, Windows Registry, and EVTX event logs from raw disk images — including EWF/E01 forensic formats.

Timeline Correlation

Reconstructs a unified forensic timeline by cross-referencing timestamps from multiple artifact sources to detect inconsistencies.

Evidence Scoring

Assigns each file a forgery likelihood score (0–100) based on the number and severity of detected timestamp anomalies.

Report Generation

Produces court-ready PDF and HTML forensic reports including case metadata, SHA-256 integrity hash, findings, and verdict.
